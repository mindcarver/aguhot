# AGUHOT — Experience Spine

## Foundation

AGUHOT 是一个响应式 Web 产品，首发面向桌面浏览器与移动端浏览器。`DESIGN.md` 是视觉身份参考；本文定义交互、信息架构、状态和阅读节奏。[ASSUMPTION] V1 的主使用场景是桌面开屏和移动端临时追踪，而不是深度移动原生操作，因此优先保证桌面阅读效率，再向移动端收敛。

[ASSUMPTION] V1 不强制登录才能浏览核心内容。登录或轻账户主要服务 `关注列表` 和后续留存能力，不应成为首次价值到达的门槛。

## Information Architecture

| Surface | Reached from | Purpose |
|---|---|---|
| 首页 / 热点事件流 | App open / 导航首项 | 扫描当日最重要的 `热点事件` |
| 热点事件详情页 | 首页卡片 / 搜索结果 / 日报 / 主题页 | 理解单个 `热点事件` 的解释、`证据时间线` 与 `市场反应信号` |
| 日报页 | 主导航 | 快速复盘一个交易日的重点 `热点事件` |
| 主题页 | 主导航 / 热点事件详情页 | 跟踪持续主线或长期主题下的多个 `热点事件` |
| 搜索结果页 | 全局搜索 | 回到某个 `热点事件`、`主题页` 或关键词 |
| 关注列表 | 主导航 / 收藏动作 | 查看已收藏的 `热点事件` 与 `主题页` |
| 运营复核台 | 内部入口 | 对 `热点事件` 归组、解释和来源做人工复核 |

[ASSUMPTION] 桌面端采用左侧导航，移动端改为顶部导航加抽屉。导航层级不超过一级；任何详情页都必须能直接返回上一级消费场景。

→ 当前这版 UX 还没有视觉 mockups；后续如果需要视觉参考，建议优先补：首页、热点事件详情页、日报页。

## Responsive & Platform

| Breakpoint | Behavior |
|---|---|
| `≥ 1200px` | 左侧导航固定可见；首页为单列主流 + 右侧轻摘要/筛选区或顶部筛选条；详情页采用宽阅读栏。 |
| `768px - 1199px` | 左侧导航收窄或折叠；首页和详情页保持单主列，筛选浮到顶部。 |
| `< 768px` | 导航进入抽屉；首页卡片纵向堆叠；详情页中的 `市场反应信号` 与关联区块依次下沉。 |

移动端不是桌面端的压缩截图。移动端必须优先保住三件事：当前热点扫描、`证据时间线` 可读、返回路径清晰。

## Voice and Tone

这里规定的是微文案，不是品牌语气。品牌姿态见 `DESIGN.md`。

| Do | Don't |
|---|---|
| “为什么重要” | “重磅利好” |
| “当前仍不确定” | “即将爆发” |
| “市场已出现反应” | “主力已进场” |
| “AI 生成摘要，已提供来源” | “AI 帮你找到发财机会” |
| “暂无可确认来源” | “小作文流传中” |

产品语气必须克制、说明式、可核查。任何会让产品滑向荐股、情绪煽动或内幕暗示的话术都应该被禁掉。

## Trust & Provenance Patterns

这是 AGUHOT 独有的关键体验层，优先级不低于首页本身。

- 每个 `热点事件` 都必须显式展示其 `证据源` 数量、最近更新时间和来源可追溯入口。
- 解释内容与事实内容必须分区，不允许视觉上混成一个大段落。
- AI 生成的摘要、翻译或解释必须有统一标识，视觉使用 `{components.ai-label}`。
- 当一个判断仍不确定时，必须允许页面保留“不确定”状态，而不是强行补全。
- 如果 `证据源` 缺失、失效或质量过低，页面应降级展示，而不是静默隐藏风险。

## Component Patterns

行为定义在这里；外观定义见 `DESIGN.md.Components`。

| Component | Use | Behavioral rules |
|---|---|---|
| 热点卡片 | 首页 / 列表 | 卡片点击默认进入 `热点事件详情页`；卡片中的标签、来源、时间等附属元素可单独交互，但不能打断主路径。 |
| 筛选胶囊 | 首页 / 日报 / 主题页 | 单击切换；激活状态明确；清除路径始终可见。 |
| 证据时间线行 | 热点事件详情页 | 每行至少可展开原始摘要并跳转来源；默认按时间顺序阅读。 |
| 市场反应 chip | 热点事件详情页 / 主题页 | 只表达一个信号维度；不承载长文案；颜色只表示市场语义。 |
| AI 标识 | 热点事件详情页 / 日报 | 所有 AI 生成内容共用一套标识规则，避免同页出现不同说法。 |
| 收藏动作 | 热点卡片 / 详情页 / 主题页 | 一次点击即可完成；需要登录时应延迟触发登录，而不是先卡住浏览。 |
| 运营复核状态标记 | 运营复核台 | 明确区分待复核、已复核、需下线；对公开展示结果有直接约束。 |

## State Patterns

| State | Surface | Treatment |
|---|---|---|
| 冷启动加载 | 首页 | 先显示热点卡片骨架，不显示假数据；骨架数量接近真实列表密度。 |
| 当日无热点 | 首页 | 文案明确说明“当前暂无可公开展示的热点事件”，同时保留最近更新与回退入口。 |
| 证据不足 | 热点事件详情页 | 保留事件摘要，但明确标记“来源不足 / 仍待确认”。 |
| 市场信号缺失 | 热点事件详情页 | `市场反应信号` 区块展示缺失说明，而不是直接消失。 |
| 搜索无结果 | 搜索结果页 | 提示无结果，并建议回到首页或切换关键词。 |
| 关注列表为空 | 关注列表 | 说明该页用途，并提供返回首页或探索主题入口。 |
| 原始链接失效 | 热点事件详情页 | 该条 `证据源` 保留时间和来源，但标注链接不可访问。 |
| 事件下线 | 热点事件详情页 / 运营复核台 | 对外页显示“该事件已撤下或暂停展示”，对内保留原因。 |

## Interaction Primitives

- **首页扫描优先。** 用户进入首页后，第一屏必须能完成“今天在交易什么”的快速判断，不要求立即点击。
- **点击主路径优先。** 热点卡片整卡可点；局部元素交互不能把用户困在局部细节里。
- **返回路径稳定。** 从 `日报`、`主题页` 或搜索进入详情后，返回时应回到原消费语境，而不是总回首页。
- **滚动是默认阅读方式。** 不使用横向 carousel 承载核心热点。
- **浮层只承载短任务。** 筛选、登录提示、轻量来源预览可以浮层化；核心解释、时间线、主题跟踪不能塞进抽屉或对话框。
- **禁止：**
  - 自动播放或强干扰动画
  - 夸张的价格闪烁和情绪化提示
  - 连续多层弹窗
  - 只靠颜色区分关键状态

## Accessibility Floor

- 所有交互元素必须可键盘访问，尤其是首页卡片、搜索、筛选、收藏和详情页来源链接。
- 结构化标题层级必须清晰，让屏幕阅读器能区分“热点标题”“版块标题”“证据时间线”。
- 红绿市场语义不能只靠颜色表达，必须辅以文本或符号标签。
- 移动端点击热区至少符合常规触控尺寸，避免密集小标签难以点击。
- `证据时间线` 中的时间、来源、摘要、链接顺序必须遵循自然阅读顺序。
- 如果用户启用减少动态效果，所有状态变化应改为即时切换而非淡入淡出。

## Inspiration & Anti-patterns

- **借鉴：AI HOT 的导航清晰度。** 左侧结构和“精选/日报/主题/收藏”这种消费路径是合理的，但 AGUHOT 需要更强的 A 股语义和更重的证据层。
- **借鉴：机构终端的可追溯意识。** 不是复制 Wind/Choice/iFinD 的全量能力，而是借用它们对来源、时间和工作流连续性的重视。
- **拒绝：交易软件式满屏红绿。** AGUHOT 的任务不是模拟盘口，而是解释热点。
- **拒绝：资讯门户式无限堆内容。** 如果首页像文章流，用户就失去事件视角。
- **拒绝：营销型 AI 体验。** “AI 帮你抓牛股”“一键看透主力意图”一类表达会直接破坏信任边界。

## Key Flows

### Flow 1 — 陈明开盘前 10 分钟扫主线

1. 陈明在桌面浏览器打开 AGUHOT。
2. 首页直接展示当日 `热点事件流`，顶部筛选默认为“今日 / 全部热点”。
3. 他快速扫过前 5 条 `热点事件`，重点看标题、一句话解释、来源数和最近更新时间。
4. 他点开最热的一条事件，进入 `热点事件详情页`。
5. 页面首屏先给出“发生了什么 / 为什么重要 / 当前仍不确定什么”，下面接 `证据时间线` 和 `市场反应信号`。
6. **Climax:** 陈明在 3 分钟内确认今天最强主线是什么，证据是否站得住，以及市场是否已经开始响应。
7. 他返回首页，继续浏览剩余热点，带着清晰优先级离开。

失败分支：如果该事件来源仍不足，详情页必须明确显示“仍待确认”，而不是把解释写得像确定事实。

### Flow 2 — 林瑶盘中追溯突然拉升的题材

1. 林瑶在盘中看到某个概念快速异动，打开 AGUHOT 并搜索对应关键词。
2. 搜索结果返回相关 `热点事件` 和 `主题页`。
3. 她进入某个 `热点事件详情页`，先看事件摘要，再沿 `证据时间线` 阅读最早来源。
4. 她对照 `市场反应信号`，判断这次异动是新闻触发、政策刺激还是情绪延续。
5. 她点击原始来源确认表达，同时把该事件加入 `关注列表`。
6. **Climax:** 林瑶拿到一条可以引用、可追溯、可继续写作的解释链，而不是只看到“涨了所以找理由”。

失败分支：如果原始链接失效，该 `证据源` 仍应保留来源和时间，并明确告知链接不可访问。

### Flow 3 — 阿周收盘后做 10 分钟复盘

1. 阿周在收盘后进入 `日报` 页面。
2. 他先浏览当日核心摘要，再依次打开 2 到 3 条最关键的 `热点事件`。
3. 对于持续发酵的题材，他跳转到相应 `主题页` 看过去几天事件的连续变化。
4. 如果某条主线值得第二天继续跟踪，他将对应 `主题页` 加入 `关注列表`。
5. **Climax:** 阿周在 10 分钟内完成一份结构化复盘，并明确第二天要继续看哪些主线。

失败分支：如果 `日报` 尚未生成完毕，页面应显示生成时间和当前覆盖范围，而不是空白页。
