## What Went Well

1. **ui-ux-pro-max 对账前置**——correct-course 阶段先跑 `--design-system`，印证 token 正确（Banking navy + gold ≈ aguhot 现有 token），把 scope 收死在「视觉处理方式」而非 token/architecture churn。这一步避免了巨大的 scope 蔓延。
2. **Demo-first**——先做 `demo-ui-redesign.html`（用真实 token 渲染），跟用户视觉迭代（方案 A/B/C → 锁 A），再动代码。用户对文字提案 pushback（「文字太抽象了」「你不就给了我一个方案吗」），看到浏览器实物才 fully engage。**UI 决策留给能展示的时候，不要描述。**
3. **Codex review per story**——每 story 1-2 轮，抓到真实问题（SearchBox 溢出 P1、timeline.spec flex 断言 P1、SourceChipList 缺 chips P1、fold +N 记录/来源语义、heading 层级、ol semantics、caller space-y-3）。Codex 的 file:line 证据 + 代码级审查比纯文字 review 有效。
4. **Scope invariant 全程守住**——token + architecture 零 diff（git diff 校验），是 Epic 6「Moderate 不升 Major」的核心依据。每个 story 的 Dev Agent Record 都记录了该校验。
5. **4.2 逻辑保留**——fold sibling pattern（`<details>` 不嵌于 `<a>`）、null guards（NFR-2）、honest-state 全部 carry forward，视觉重构没破坏行为契约。

## What Didn't Go Well / Challenges

1. **e2e 全程未跑**——无 `DATABASE_URL`，所有 e2e（home/timeline/search/navigation/a11y/design/themes）deferred。视觉用 scratch + typecheck/lint/prettier 验证，但 e2e 断言是 blind 更新（逻辑正确，执行未验证）。**这是最大的 loose end**——production deploy 前必须在 DB 环境跑全 e2e。
2. **Dev server 不稳定**——Next/Turbopack 多次崩（`SyntaxError: Unexpected end of JSON input`），打断视觉验证流。重启若干次。
3. **Spec Code Map 偏差**——6.1 spec 写「新建 top-nav.tsx」但实际原地重构 PublicNav（避免重复 ~150 行，documented）；6.4 SourceChipList 最初只渲染 count（偏离 spec 的「关联讨论 N 条 + chips」）——Codex 抓到。**教训：spec AC 是契约，偏离要么显式记录要么会被 review 抓。**
4. **Scratch 路由命名**——`__timeline-preview` 404（Next.js 把 `_` 前缀文件夹当私有，不路由）。小坑，但费了一轮。
5. **timeline.spec 反复改**——6.2/6.3/6.4 各自打破 timeline.spec 断言（band/UTC/dl/fold），per-story 修。本可批量改 e2e，但 per-story 是 Codex 驱动的纪律（不留 guaranteed-fail）。合理但 churn 多。

## Key Insights

1. **「形态可借，护城河不能丢」**——借了参考站的视觉形态（无边框编辑型纵栏），但保留 aguhot 的可信度护栏：证据可追溯（海军蓝竖线语义）、AI 标识 + 「AI 解读」命名（不回退「推荐理由」）、NFR-2 不造假精选分（用「来源 N」chip）、红绿仅市场语义。视觉对齐没有妥协产品的信任层。
2. **视觉决策需要视觉验证**——文字 UI 提案被拒，渲染 demo（真实 token）才 engage。以后 UI 活先出浏览器 demo。
3. **deferred-e2e 模式是系统性验证缺口**——Epic 4/5/6 都因无 DB 而 defer e2e。typecheck/lint/prettier + scratch 视觉对代码正确性 solid，但 e2e 执行是缺失的验证门。需要在 DB 环境补跑。

## Action Items

1. **[HIGH] 在 DB 环境跑全 e2e**——Epic 6（及 4/5）的 e2e 全 deferred。home/timeline/search/navigation/a11y/design/themes 需在有 `DATABASE_URL` 的环境跑一遍，验证视觉重构没回归。Owner: dev/infra。
2. **[HIGH] 建立 local DB / CI DB**——deferred-e2e 是系统性问题（Epic 4/5/6 都中招）。需要一个可用的 local PG（aguhot_dev）或 CI DB，让 e2e 能跑。Owner: dev/infra。
3. **[MED] 排查 dev server 崩溃**——Next/Turbopack `SyntaxError: Unexpected end of JSON input` 多次出现。查根因；考虑非 Turbopack dev 或更稳定配置。Owner: dev。
4. **[LOW] 设计系统 showcase**——/design 页已存在。把 Epic 6 新组件（TopNav / NumberedHotList / DateSectionDivider / SourceChipList / EditorialReasonBlock）+ 无边框编辑型纵栏模式 formalize 成 /design 的文档化条目。Owner: UX/dev。
